## 1. 앱 개요

세트 쉬는 시간(60~90초)에 한 손으로 입력 완료할 수 있는 극도로 단순한 웨이트 트레이닝 기록 PWA 웹앱.
데이터는 LocalStorage 우선 저장 + GAS(Google Apps Script) 시트 동기화.
기존 이름표 프로젝트와 동일한 기술 스택(단일 HTML + JS + CSS).
모바일 전용 — 데스크톱 레이아웃 고려하지 않음.

---

## 2. 사용 흐름

1. 체육관 도착 → 앱 열기
2. 메인 화면에서 **운동 시작** 버튼 탭
3. 부위 태그 선택 (하체, 어깨, 가슴, 등, 데일리, 인터벌 — 탭한 순서대로 정렬)
4. 첫 태그 선택 시 운동 타이머 자동 시작
5. 현재 종목 카드 1개만 펼쳐짐 → 세트별 무게/횟수 입력 → 체크
6. 체크 시 즉시: (a) 해당 세트 저장 (b) PR 판정 (c) 휴식 타이머 시작
7. 하단 종목 버튼으로 다음 종목 전환
8. 전종목 완료 → **운동 완료** 요약 카드 표시
9. 메인 화면으로 복귀 — 캘린더와 통계에 즉시 반영

## 3. 화면 구조

### 변경 이력
- v1: 3탭 구조 (홈 / 운동 / 통계) + 하단 탭바
- **v2 (현재)**: 싱글 화면 구조. 하단 탭바 제거. 메인 화면 하나에서 기록·통계·운동 시작 전부 처리.

### 3-1. 메인 화면 (유일한 고정 화면)

위에서 아래 순서:

**(a) 헤더 (`div.app-header`)**
- 좌측: 햄버거 메뉴 아이콘 (설정/인바디 진입용 — 2차 구현)
- 중앙: `◀ 3월 ▶` 월 전환 버튼
- 우측: 월간 캘린더 팝업 토글 아이콘 (달력 SVG)

**(b) 요약 메시지 (`div.summary-msg`)**
- 큰 텍스트: "3월에는 **0kg** 들었어요"
  - 해당 월 전체 totalVolume 합산
  - 데이터 없으면: "아직 운동 기록이 없습니다"
- 작은 텍스트 (피드백):
  - 데이터 없을 때: "첫 운동을 시작해보세요!"
  - 데이터 있을 때: "지난달 대비 +15%" 또는 "이번 주 3회 운동, 잘하고 있어요"
  - 로직: 전월 동기간 대비 볼륨 증감률 계산. 운동 횟수 기반 격려 문구.

**(c) 히어로 카드 (`div.hero-card`) — 2차 구현**
- 데이터 없을 때: 숨김 또는 "첫 운동을 시작해보세요!" 안내
- 데이터 있을 때:
  - 가장 많이 한 부위 + 총 볼륨 (예: "하체 4,800kg")
  - 하위에 종목별 요약 2열 그리드 카드

**(d) 주간 캘린더 (`div.week-cal`)**
- 이번 주 일~토 가로 배치 (7칸)
- 오늘 날짜: 블루 원형 배경 하이라이트
- 운동한 날: 날짜 아래 작은 도트 (부위 색상 — 모노톤 적용 후에는 `#2D2D2D` 단일 도트)
- 주간 캘린더에서 과거 날짜 클릭 → 아래 영역에 해당일 운동 요약 표시

**(e) 월간 캘린더 (`div.month-cal`)**
- 기본 펼침 상태
- 운동한 날에 도트 표시
- 날짜 클릭 → 바텀시트 팝업 (`div.bottom-sheet`)
  - 바텀시트 내용: 해당일 운동 요약 (날짜, 부위 태그, 종목 리스트, 세트수, 총 볼륨, 시간)
  - 하단에서 슬라이드업 애니메이션
  - X 버튼 또는 바깥 영역 탭으로 닫기

**(f) 운동 시작 버튼 (`button.start-workout`)**
- `position: fixed; bottom: 20px; left: 20px; right: 20px`
- 높이 52px, 배경 `#2D2D2D`, 흰 텍스트 "운동 시작", `border-radius: 12px`
- 탭 시 → 운동 진행 화면으로 전환

### 3-2. 운동 진행 화면 (운동 시작 후 표시)

메인 화면을 숨기고 운동 화면을 표시. 완료/취소 시 메인으로 복귀.

**(a) 상단 바**
- 좌측: 운동 경과 시간 (mm:ss, 실시간 카운트업)
- 중앙: 선택된 부위 태그들 (가로 나열, 현재 부위 하이라이트)
- 우측: "완료" 버튼

**(b) 현재 종목 카드 (1개만 펼침)**
- 종목명 + 부위 색상 사이드바
- "오늘 0kg (지난번 240)" — 실시간 볼륨 비교
- 세트 테이블:
  - 헤더: 세트 | kg | 횟수 | 완료
  - 각 행: 세트번호 | 무게 입력 | 횟수 입력 | 체크 버튼
  - 지난번 값: 입력 필드 아래 연한 회색으로 표시
  - 체크 버튼만 탭 → 지난번 값으로 자동 채움 (1탭 완료)
  - 값 수정 시 → 숫자 수정 후 체크 (2~3탭)
  - **워밍업(W) 토글 없음** (v2에서 제거)
- "+ 세트 추가" 버튼
- PR 감지 시: 해당 세트 행 배경 플래시 + "🏆 개인 기록!" 인라인 표시

**(c) 종목 탐색 버튼바 (`div.exercise-nav`)**
- 카드 아래 가로 스크롤 버튼 영역
- 각 버튼: 종목 이름 (짧게)
- 현재 종목: 활성 스타일 (배경 `#2D2D2D`, 흰 텍스트)
- 완료된 종목: 체크 아이콘 + 흐린 스타일
- 미완료 종목: outline 스타일
- 버튼 클릭 → 해당 종목 카드로 전환

**(d) 휴식 타이머**
- 세트 체크 시 자동 카운트다운 (기본 90초, 종목별 설정 가능)
- `Date.now()` 기반 — 종료 시각 저장, 화면 복귀 시 남은 시간 재계산
- 큰 숫자로 카운트다운 표시
- 0 도달 시 색상 변화 (예: 파란→빨간)
- 소리/진동 없음

**(e) 운동 완료 요약 카드**
- "완료" 버튼 탭 시 오버레이로 표시
- 총 운동 시간
- 총 볼륨 (kg) + 지난 동일 부위 조합 대비 ± 표시
- 추정 칼로리
- PR 갱신 목록 (종목명, 이전→현재)
- 총 종목 수 / 총 세트 수
- "확인" 버튼 → 메인 화면 복귀

### 3-3. 바텀시트 팝업 (캘린더 날짜 클릭 시)

- `position: fixed; bottom: 0` 에서 슬라이드업
- 상단: 날짜 + X 닫기 버튼
- 내용: 해당일 세션 요약 (부위 태그, 종목 리스트 + 세트수, 총 볼륨, 시간)
- 운동 안 한 날: "운동 기록이 없습니다"
- 배경 딤(반투명 검정) 처리

### 3-4. 설정 (2차 구현)

- 햄버거 메뉴에서 진입
- 종목 편집 (이름, 부위, 기본 세트/횟수/휴식시간)
- 부위별 종목 순서 변경
- 체중 수동 입력 (칼로리 계산용)
- 인바디 기록 입력/조회

## 6. 칼로리 추정

`칼로리(kcal) = MET × 체중(kg) × 시간(시간) × 1.05`

- 종목별 MET 값은 종목 마스터에 정의
- 체중: 인바디 기록 최신값 참조. 없으면 기본값 70kg.
- 정밀도가 목적이 아님 — 피드백/동기부여용 추정치

### 8-2. 기술 스택
- HTML5 + CSS3 + Vanilla JS (프레임워크 없음)
- LocalStorage (1차 데이터 저장)
- Google Apps Script (2차, 시트 동기화)
- PWA: manifest.json + Service Worker (2차)

### 8-3. 웹앱 제약사항 (iOS Safari)
- **진동 불가**: Vibration API 미지원. 시각 표시만 사용.
- **백그라운드 타이머 제한**: `setInterval`이 백그라운드에서 멈춤. `Date.now()` 기반 종료 시각 저장, 포그라운드 복귀 시 남은 시간 재계산.
- **소리 제한**: 사용자 인터랙션 없이 자동 재생 불가. 1차에서 소리 미포함.

## 9. 시각화 계획 (2차)

### 9-1. 히어로 카드
- 해당 월 가장 많이 한 부위 + 총 볼륨
- 하위 종목 2열 그리드 (종목명 + 볼륨)
- 가계부 앱의 "이번 달 가장 많이 쓴 곳" 패턴 차용

### 9-2. 트리맵 / 버블 차트
- 부위별 볼륨 비중을 면적으로 시각화
- 가슴 30%, 하체 25%, 등 20% 등

### 9-3. 월별 바 차트
- 최근 6개월 총 볼륨 막대 그래프
- 현재 월 강조 색상

### 9-4. 인바디 추이 그래프
- 체중 / 체지방률 / 골격근량 시계열 라인 차트
- 측정 포인트만 점으로 표시
